客户端登录流程设计

主播端（VideoRecoder）和观众端（MediaPlayer）都需要登录，但角色不同

场景对比

主播端 — VideoRecoder

工作流程

  1. 启动 → 弹出登录窗口
  2. 登录/注册（角色=主播）
  3. 获取推流密钥（服务器生成）
  4. 开始录制 + 推流到 RTMP 地址
  5. 观众进入直播间

核心区别：登录后需要申请"推流密钥"才能推流

观众端 — MediaPlayer

工作流程

  1. 启动 → 弹出登录窗口
  2. 登录/注册（角色=观众）
  3. 获取直播间列表
  4. 选择一个直播间 → 拉流播放

核心区别：登录后浏览直播间列表，选择直播间拉流观看

关键交互问题

A. 分开应用 + 各自内嵌登录

VideoRecoder 和 MediaPlayer 各加一个登录对话框，角色在注册时选择（主播/观众），登录后进入对应功能

改动最小，两个应用各自独立

B. 统一启动器 + 两个子应用

新建一个 Launcher 程序负责登录，登录后根据角色启动 Recorder 或 Player

架构更清晰，但多一个进程需要维护

推荐方案A —— 改动最小，在两个现有应用中各加一个登录对话框即可。请选择。
